WHSParameters_PutLocationDirectiveLockTime=Geben Sie die maximale Zeitspanne ein, die das System warten soll, um eine Lagerplatzsperrung bei der Verarbeitung von Lagerplatzrichtlinien für Einlagerungsarbeiten zu aktivieren./r/nDiese Einstellung ist relevant, wenn mehrere Threads versuchen, Lagerplatzrichtlinien für Einlagerungsarbeiten gleichzeitig auszuführen. Wenn ein Thread versucht, einen Einlieferungslagerplatz zu finden, führt er mehrere Prüfungen durch, um sicherzustellen, dass der Lagerplatz für die Einlagerungsarbeit geeignet ist. Einige Überprüfungen (z. B. Mischungseinschränkungen) müssen Thread für Thread verarbeitet werden, um zu verhindern, dass mehrere Threads versuchen, denselben Lagerplatz zu verwenden. Wenn zusätzliche Threads versuchen, denselben Lagerplatz zu überprüfen, werden diese Threads blockiert und in die Warteschlange gestellt. Diese Einstellung stellt die maximal akzeptable Zeitspanne dar, die ein Thread auf die Freigabe der Sperrung wartet, nachdem er von einem vorherigen Thread für denselben Lagerplatz erworben wurde. Maximal 30 Sekunden (30.000 Millisekunden) sind in der Regel eine gute Wahl. Wenn die Wartezeit zu kurz ist, riskieren Sie, Arbeit mit einer großen Anzahl verschiedener Einlieferungslagerplätze zu produzieren. Eine relativ lange Wartezeit verschwendet normalerweise keine Ressourcen, denn sobald eine Sperre freigegeben wird, wird der nächste Vorgang sofort fortgesetzt. Die Wartezeit sollte trotzdem nicht zu lang sein, da eine lange Verzögerung auf ein anderes Problem hinweisen kann. Wenn der Thread also fehlschlagen kann, kann ein Zurücksetzen erfolgen, oder Sie können das Problem beheben und so übermäßige Verarbeitungszeiten vermeiden, die durch längere Zeit feststeckende Threads entstehen können.\r\n

WHSParameters_WHSParameters_WaveLockWait=Geben Sie die maximale Wartezeit für den Erwerb einer Objektsperre (plus Dimension) während der Wellenverarbeitung ein.\r\nDiese Einstellung ist relevant, wenn mehrere Wellen hintereinander freigegeben werden. Aufgrund der Überlastung beim Zugriff auf die Tabellen werden jedoch einige Wellen gleichzeitig ausgeführt. Wenn eine Welle verarbeitet wird, versucht sie während des Zuordnungsschritts, eine Sperre für die Kombination von Element plus den Abmessungen über der Position zu erhalten. Wenn eine oder mehrere gleichzeitige Wellen auch versuchen, dieselbe Sperre für einen identischen Datensatz zu erhalten, werden diese Wellen blockiert und in die Warteschlange gestellt. Der Wert gibt die maximal zulässige Zeit an, die eine Welle darauf wartet, dass eine Sperre freigegeben wird, wenn die Sperre von einer vorherigen Welle für dasselbe Element plus Dimension über der Position erfasst wurde. Es wird eine Staging-Periode zwischen den Wellen festgelegt. Maximal 60 Sekunden (60.000 Millisekunden) sind normalerweise eine gute Wahl. Wenn die Wartezeit zu kurz ist, besteht die Gefahr, dass aufgrund von Zeitüberschreitungen ein hohes Volumen an Wellenfehlern auftritt. Eine relativ lange Wartezeit verschwendet normalerweise keine Ressourcen, da der nächste Vorgang sofort fortgesetzt wird, sobald eine Sperre aufgehoben wird. Sie sollten es jedoch nicht zu lange warten lassen, da eine lange Verzögerung auf ein anderes Problem hinweisen kann. Wenn Sie also zulassen, dass die Welle ausfällt, wird möglicherweise etwas zurückgesetzt oder Sie können das Problem beheben, wodurch die übermäßigen Verarbeitungszeiten vermieden, wenn Wellen für zu lange Zeit zurückgesetzt werden.
